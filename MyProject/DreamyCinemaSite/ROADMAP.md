# Dreamy Cinema 最终功能路线图

## 1. 产品目标

构建一个运行在个人 Windows 电脑上的私有视频片库：视频保存在本机磁盘，单个管理员通过电脑或同一局域网内的手机完成同步、整理、筛选、AI 字幕翻译和播放。

最终版本的核心边界：

- 单管理员，不做多用户权限体系。
- 视频通过 `Videos` 文件夹和“同步”按钮导入，不做网页上传。
- 默认只允许本机和可信局域网访问，不直接暴露到公网。
- 以 MP4 浏览器直播放为基准，不默认转码或复制出多份视频。
- 暂不保存播放进度，也不根据播放行为自动设置已看/未看。
- AI 字幕属于核心功能；支持已有字幕翻译，无字幕时可先进行语音识别再翻译。

## 2. 已完成的核心能力

### 视频入库

- 扫描 `Videos` 顶层 MP4，并显式同步。
- 按年月移动到 `Videos/originals/yyyy/MM`。
- SQLite 保存视频记录，文件丢失时标记为缺失。
- ffprobe 提取时长、分辨率和编码。
- FFmpeg 自动截取中间帧作为封面。
- 支持手动替换、移除和重新同步生成封面。

### 整理与检索

- 唯一番号、标题、简介和创建日期。
- 分类标签及视频/标签多对多关系。
- 标签分类和标签的新增、重命名与删除。
- 多标签交集筛选。
- 番号、标题、简介和原文件名搜索。
- 日期、番号、大小和同步时间排序。
- 服务端分页、手机无限加载和封面懒加载。

### 播放与维护

- 列表内播放和独立播放页。
- HTTP Range，支持拖动视频进度条。
- 针对小米浏览器原生视频层的页面结构适配。
- 回收站、恢复、永久删除和缺失记录清理。

### 安全与架构

- Vue 3、TypeScript、Vite、Pinia 和 Vue Router 前端。
- ASP.NET Core、EF Core 和 SQLite 后端。
- 单管理员密码、Cookie 登录、CSRF 和登录限流。
- 首次密码只能在服务器本机设置。
- 局域网监听和仅限专用网络/本地子网的 Windows 防火墙规则。
- 移动端 Playwright 端到端测试。

## 3. 最终版必做阶段

### 阶段 A：字幕基础与播放器集成

目标：每个视频可以管理并播放一个或多个原文、译文或双语字幕轨道。

- 建立 `SubtitleTracks`、`SubtitleCues` 和字幕文件的数据模型。
- 同步时识别同名 `.srt`、`.vtt` 和 `.ass` 外挂字幕。
- 使用 FFmpeg 检测并提取视频内嵌字幕轨道。
- 字幕文件保存在 `Videos/subtitles/{videoId}`，数据库记录语言、来源、格式和状态。
- 统一解析为带稳定 Cue ID 的内部结构，时间轴与文本分离保存。
- 播放器支持关闭、原文、译文和双语轨道切换。
- 为浏览器输出 WebVTT，同时保留原始字幕文件。

验收：导入带字幕的视频后，电脑和手机播放器都能切换并正确显示字幕，时间轴不被改变。

### 阶段 B：可恢复的后台任务框架

目标：同步、语音识别和 AI 翻译都作为可观察、可取消、可恢复的后台任务执行。

- 同类任务只允许一个实例处理同一视频，避免重复生成。
- 保存任务类型、阶段、进度、当前文件、错误、重试次数和时间。
- 页面可以离开后重新进入并继续查看任务进度。
- 支持取消、失败重试和服务重启后的安全恢复。
- 同步任务逐文件执行移动、分析、封面和字幕发现。
- AI 任务支持按字幕分块断点续传，已完成分块不重复计费。

验收：长视频处理期间刷新或关闭页面不影响任务；重复点击不会创建冲突任务。

### 阶段 C：AI 字幕识别与翻译

目标：把已有外语字幕翻译为中文；没有字幕时可先从音频生成原文字幕。

- 有原文字幕时直接翻译，不重复做语音识别。
- 无字幕时使用可配置的 Whisper 兼容语音识别引擎生成带时间轴的原文。
- AI 翻译提供可替换的 Provider 接口，不把业务逻辑绑定到单一厂商。
- 配置源语言、目标语言、模型、API 地址和密钥；密钥不写入日志或备份包。
- 按上下文分块翻译，但使用稳定 Cue ID 保证返回结果与原时间轴一一对应。
- 支持专有名词词汇表、人物名统一和翻译风格设置。
- 校验缺行、多行、空译文、编号错位和时间轴修改，异常分块自动重试。
- 记录 token/字符用量、耗时和失败原因，并允许设置单任务用量上限。

验收：同一字幕翻译前后 Cue 数量和时间轴完全一致，中文轨道可播放，失败后可以从未完成分块继续。

当前状态：阶段 31 的真实本地 Provider 已实现并在 RTX 3060 Ti 上通过隔离验证。faster-whisper 与 llama.cpp 仅监听回环地址；识别/翻译串行执行；翻译严格校验 Cue ID；应用重启、运行中取消和分块恢复均已验证。600 秒隔离基准已通过，剩余验收是加入真实片源后的手机人工观看，随后进入字幕校对/润色阶段。

### 阶段 D：字幕校对、双语与导出

目标：管理员可以在手机或电脑上检查和修正 AI 结果，而不是只能接受整份机器翻译。

- 原文与译文按时间轴对照显示。
- 支持逐条编辑、搜索、跳转视频时间和保存。
- 支持选择部分 Cue 重新翻译，不覆盖已人工修改内容。
- 标记未翻译、疑似错位、过长和阅读速度过快的字幕。
- 生成中文和双语字幕轨道。
- 导出 UTF-8 SRT、WebVTT；ASS 在明确样式保留规则后支持导出。
- 播放器记住当前设备最近选择的字幕语言，但不保存播放进度。

验收：人工修改一条译文后重新翻译其他区段，该修改保持不变；导出的字幕可以被常见播放器读取。

### 阶段 E：数据备份

目标：片库整理结果可以独立于视频原文件安全保存。

- 使用 SQLite 在线备份 API 创建一致性快照。
- 备份数据库、封面、字幕、词汇表、必要配置和版本清单。
- 默认不打包体积较大的 `Videos/originals`。
- 生成带时间戳的 ZIP，并记录大小、校验值和创建时间。
- 网页支持创建、下载和删除备份。

验收：服务运行且正在使用数据库时仍能生成可校验的备份包，不影响视频播放。

### 阶段 F：恢复与灾难保护

目标：数据库损坏、误操作或迁移电脑后可以恢复片库。

- 上传或选择本地备份包进行恢复。
- 校验清单、文件路径、数据库结构和校验值。
- 恢复前自动备份当前状态。
- 安全替换数据库和封面，失败时回滚。
- 恢复后检查视频文件是否存在，并刷新缺失状态。

验收：在隔离环境完成一次“备份 → 修改数据 → 恢复”，标题、标签、封面、字幕和人工校对内容全部还原。

### 阶段 G：批量整理与重新处理

目标：视频数量增加后仍能高效维护元数据。

- 多选视频并批量添加或移除标签。
- 批量移入回收站。
- 单个或批量重新提取媒体信息。
- 单个或批量重新生成自动封面。
- 根据分辨率建议清晰度标签，由管理员确认后应用。
- 对浏览器不兼容的编码给出明确警告，不静默播放失败。

验收：管理员可以在手机上用一次操作为一组视频调整标签，并能重试失败的媒体分析。

### 阶段 H：账户与安全维护

目标：管理员可以长期维护登录凭据并清楚知道当前访问边界。

- 登录后修改管理员密码。
- 修改密码后使现有登录会话失效。
- 提供明确的登录过期和重新登录处理。
- 对写操作、文件路径和备份恢复继续执行权限与 CSRF 校验。
- 记录关键管理操作日志，不记录密码和敏感 Cookie。

验收：密码修改后旧 Cookie 不能继续写入数据，未登录请求不能读取片库 API。

### 阶段 I：Windows 正式部署

目标：服务器电脑重启后网站可自动恢复运行，不依赖手动打开终端。

- 提供 `dotnet publish` 发布流程和独立生产配置。
- 作为 Windows 服务或受控启动任务运行。
- 增加 `/health`，检查进程、数据库和视频目录可用性。
- 结构化滚动日志和可诊断的启动错误。
- 明确本机地址、局域网地址、端口和防火墙状态。
- 局域网 HTTP 作为默认部署；远程访问只通过 HTTPS 反向代理或可信 VPN。

验收：重启 Windows 后无需登录桌面即可访问网站；数据库或视频目录异常时健康检查能指出原因。

### 阶段 J：发布质量与验收

目标：形成可重复安装、升级和验证的稳定版本。

- 后端集成测试覆盖同步、分页、交集筛选、字幕解析/翻译校验、回收站、备份和恢复。
- 移动端 E2E 覆盖登录、筛选、编辑、字幕切换/校对、播放、批量操作和任务进度。
- 验证桌面 Edge/Chrome 和实际小米浏览器。
- 使用大量元数据记录验证分页和筛选性能。
- 运行 NuGet/npm 漏洞检查。
- 完成安装、升级、备份恢复和故障排查文档。
- 发布前执行真实备份恢复演练。

验收：自动测试通过，实际手机主流程通过，发布包可在干净目录启动并恢复备份。

## 4. 可选扩展

以下功能不阻塞局域网单人版本完成：

- MKV、MOV、WebM 等更多容器格式。
- 不兼容编码的后台转码。
- 外部网站元数据和封面抓取。
- 播放进度、自动已看和继续播放。
- PWA、桌面快捷方式和离线页面。
- 多用户、角色权限和个人收藏。
- 网页上传视频。
- 公网域名、Caddy HTTPS 或 Tailscale 远程访问。

## 5. 完成定义

最终版本完成需要同时满足：

1. 视频可以可靠同步、整理、筛选、分页浏览和播放。
2. 外挂或内嵌字幕可以导入、播放、AI 翻译、人工校对并导出。
3. 无字幕视频可以通过语音识别生成原文，再生成时间轴不变的中文字幕。
4. 元数据、标签、封面、字幕和人工校对内容可以备份并经过真实恢复验证。
5. 同步、识别和翻译等长任务不会阻塞页面或重复处理。
6. 管理员密码可以安全变更，匿名用户无法访问片库。
7. Windows 重启后服务自动启动，并具有健康检查和可用日志。
8. 自动测试、实际手机测试、依赖安全检查和发布文档全部完成。
